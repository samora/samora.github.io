Tips to those new to Git

  • Git follows the Pareto principle (80-20 rule).
  • Understand the basic Git workflow from the command line. Makes it easy to use visual Git tools (eg. TortoiseGit).
  • Avoid branching until you are comfortable with the basics.
  • Git is well documented.

Basic Git commands

  • git init: Initialize a local Git repository in current folder.
  • git add: Add stuff to staging area.
  • git commit: Commit stuff in staging area to local Git repository.
  • git status: Current status of local Git repository.
  • git remote: Manage local Git repository's remote repositories.
  • git push: Update remote with changes from local Git repository.
  • git pull: Retrieve changes from remote and update local Git repository.
  • git clone: Clone a Git repository into a new directory.
