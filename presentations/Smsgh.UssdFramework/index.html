Smsgh.UssdFramework

Easily create dynamic USSD applications

Presented by Samora Dake / @PapaSamo247

Why a new framework?

Previous USSD framework fell short in a few areas:

  • Explicit routing
  • Integrated session store
  • Step based flow
  • Unnecessary abstractions

Features

  • Automated session management.
  • Fully asynchronous.
  • Pluggable session store. RedisStore included.
  • ASP.NET MVC / Web Api -ish controllers.
